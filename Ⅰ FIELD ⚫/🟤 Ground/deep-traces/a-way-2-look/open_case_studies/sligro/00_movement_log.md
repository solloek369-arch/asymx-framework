# Movement Log — Sligro Public Source Reviews

## Why this log exists

This folder is part of an open-source AsymX visibility exercise.

Because the work happens in public, changes in direction should also be visible.

The goal is not to create a perfect first version.

The goal is to show a careful movement:
from public signal,
to claim-safe question,
to small validation pattern,
without pretending to know internal truth.

## Movement 001 — From Open Case Study to Public Source Review

### What became visible

The folder originally used the phrase open case study.

That phrase can be useful, but it may also suggest internal access, diagnosis or consultancy-style conclusions.

### Why we moved

The work is based only on publicly visible information.

Therefore the safer and more accurate framing is:

- public source review;
- public signal review;
- open visibility lens.

### What stays

The company name can remain when it refers to public sources.

The public signal can remain.

The AsymX interpretation can remain.

### What changes

The source status must become visible earlier and more often.

Each case should show:

1. what is publicly visible;
2. what is AsymX interpretation;
3. what is not being claimed;
4. what the next safe step could be.

## Movement 002 — From Claim-Safe Text to Claim-Safe Structure

### What became visible

A text can be claim-safe sentence by sentence while still creating risrough structure.

Risk can appear in:

- file names;
- folder names;
- titles;
- commit messages;
- case framing;
- missing source-status blocks.

### Why we moved

Claim safety is not only about individual sentences.

It is also about architecture.

### What changes

Every public company-facing folder needs a protocol layer.

Every case needs visible claim boundaries.

Every interpretation must stay separate from the public source.

## Movement 003 — No Panic Cleanup

### What became visible

There was a temptation to move from public use directly to damage-control language.

That is too far.

Public information is not private information.

Open-source review is allowed when source status and claim boundaries are clear.

### Why we moved

The middle path is stronger:

- do not hide;
- do not overclaim;
- do not diagnose;
- do not pretend internal access;
- do show sources;
- do show interpretation boundaries;
- do show why corrections happen.

### What stays

No history rewrite is needed unless private, nfidential or non-public information is discovered.

For now, the correct movement is visible calibration.

## Current Working Rule

If a public company name is used in an open-source AsymX case, the case must make this explicit:

This document uses only publicly visible information.

It does not use internal data.

It does not claim internal truth.

It translates public signals into claim-safe visibility questions.

## Closing Rule

Open movement builds trust when the reason for the movement is visible.
