# field-system-core — early root trace

**Status:** archived early seed / origin trace  
**Source repo:** `field-system-core`  
**Current active public field:** `asymx-framework`  
**Routing:** preserve the seed, do not continue it as a separate public system.

`field-system-core` was an early experiment in role-based friction mapping.

It explored a simple question:

> Can different roles reveal different friction patterns without immediately turning those patterns into blame, proof, or solution claims?

The useful seed was small:

- roles can act as lenses
- friction events can be grouped by context
- repeated signals can become visible before explanation
- operational drag can be observed without declaring guilt
- pattern-counting can support looking, not verdicts

The old repo used direct language such as “submit friction events” and “fork when specific.”  
In the current AsymX routing, that language is too active and too close to public claim movement.

Current AsymX adds stronger gates:

- **public = no by default**
- **source / vault first**
- **human + X gate before public movement**
- **claim-light language**
- **GitHub is a visibility layer, not the whole field**
- **AI is a lens, not a source of truth**

## Tree mechanic

This trace belongs in the tree as an early root.

The tree is used here as a functional orientation mechanic:

- root: source remains intact
- trunk: repeated signal begins to organize
- branch: role or route becomes visible
- leaf: bounded public-safe form
- compost: old form returns without being erased

This is not doctrine, proof, ownership, or sacred branding.

Tree / root / branch / leaf language is an interface for routing visibility.

## Current routing decision

`field-system-core` should not remain the active public system identity.

It may remain only as:

- archived trace
- old seed
- historical pointer
- small mechanic absorbed into the AsymX tree

The active public-gated work continues in `asymx-framework`.
