# GitHub SEO / Traffic Statistics Log

## Purpose

This log stores **GitHub SEO / repository traffic statistics** exactly as they appear in GitHub’s Traffic analytics. It supports tracking **visibility and access activity** over time without turning the numbers into proof of intent, identity, or outcome.

---

## Source

Figures are copied from:

**GitHub → Repository → Insights → Traffic**

They reflect GitHub’s own dashboard for the listed repository and window (for example, **last 14 days**). If GitHub changes labels, windows, or definitions, later entries should note that drift rather than forcing continuity.

---

## Reading Rules

- **Displayed metrics ≠ interpretation.** What GitHub prints is recorded here; conclusions stay optional and separate.
- **No visitor or cloner identification.** Counts are aggregates; this log does not infer who viewed or cloned.
- **No personal or reputational claims.** Traffic is repository-level visibility as shown by GitHub, not validation of ideas or people.
- **Screenshots / exports are the ground truth for each entry.** When recording an observation, prefer copying GitHub’s labels and numbers verbatim.
