## Rules

- **No hard claims without source** — URL + retrieval context in `01_public_source_catalog.md` (or an added row there first).  
- **No accusation language** — this layer is an **invitation to research**, not an indictment.  
- **No “solution” framing** — keep questions and visibility; avoid prescribing organisational action.  
- **No personal targeting** — discuss **public roles** and **signals**, not individuals.  
- **No private or non-public emails** — only the future **light** `pers@bol.com` gate, when the human allows it; **no scraping**, no alternate contacts documented here.  
- **No public pressure campaign** — GitHub is a **trace** and **invitation**, not leverage.  
- **All interpretations remain correctable** — prefer “appears”, “may”, “open question”, **AsymX lens / hypothesis** where needed.  
- **Bol must be able to correct, add context, or decline without friction** — no moral cornering; welcome amendments and clarifications.

## AI and numbers

- **Do not** treat AI-generated content (including summaries of reports) as verified source data.  
- If a number is not directly verified from the uploaded/public source, mark **NEEDS SOURCE CHECK** (`03_kpi_leaf_table.md`, `02_duurzaamheidsverslagen_2024_2025.md`).
