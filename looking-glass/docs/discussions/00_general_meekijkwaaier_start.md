# General Discussion — What Are We Missing?

AsymX is being developed as a public mechanism field: a place to make patterns, routing rules, safety boundaries, and correction paths visible.

This repository is not presented as final truth, proof, or a deciding source.  
It is a working public garden.

We will miss things.  
That is expected.

Ideas, corrections, clearer wording, safer boundaries, missing links, and alternative interpretations are welcome.

## What we are trying to make visible

- how signals become issues, notes, prompts, checks, or public mechanisms
- how public material can stay separate from raw/private source material
- how AI output can be treated as trace instead of a deciding source
- how public correction can happen without pressure
- how GitHub can become a shared connection layer instead of only code storage

## Useful responses

You can help by adding:

- a clearer phrase
- a missing risk
- a broken or confusing link
- a safer boundary
- a better GitHub routing idea
- an example of where the mechanism is unclear
- a question that should be answered before something becomes public

## Please do not post publicly

Please do not post incidents that belong outside the public field, unprocessed chat transcripts, identifiable personal material, sensitive internal-to-an-organization claims, or anything that would expose someone who has not chosen to be part of the public field.

If something seems important but not safe to post, describe the abstract mechanism instead.

## Routing suggestions

- Question → Discussion
- Concrete correction → Issue or Pull Request
- Safety concern → label as claim-safety / vault-boundary
- Mechanism idea → mechanism proposal
- Broken link or wording issue → small correction

## Closing

Corrections, clearer wording, and safer boundaries are welcome.

The field grows by being corrected.
