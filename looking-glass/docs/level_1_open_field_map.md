# AsymX Level 1 — Open Field Map

AsymX Level 1 tools are local-only visibility lenses.

They are small, static, inspectable demos that make one hidden pattern visible without uploading data.

Level 1 means:

- local-only
- no backend
- no account
- no tracking
- no real company data
- no hard claims
- source stays intact
- analysis is metadata
- output is temporary or exportable
- the user keeps control

The goal is not to sell a platform.

The goal is to publish small, safe entry points people can inspect, fork, test and improve.

## Not owned

This field is open by design.

It is not presented as something that must belong to one person, one company or one closed platform.

The Splitter appears where a reflective view is useful.

Everyone is welcome who is open to a clean, local, evidence-safe way of looking.

## Level 1 contract

Every Level 1 lens should follow the same minimal contract:

- one clear purpose
- one local input path
- one visible transformation
- source remains intact
- analysis is metadata
- output is exportable
- no upload
- no backend
- no hidden dependencies
- no hard claims without validation
- README explains intent, input, output and safety

Suggested map structure per lens:

- README.md
- index.html
- SECURITY.md
- docs/
- samples/ or demo data if useful
- schema/ if structured input is used
- checksums/ if useful

## Lens families

### A — Source and intake lenses

These lenses help preserve source material without immediately rewriting or summarizing it.

Possible lenses:

- `workbench/chat-intake-level1`
- `workbench/story-filter-level1`
- fractal-web-navigator-level1
- source-coordinate-level1
- trace-pad-level1

Core principle:

source first, interpretation later.

### B — Claim and trust lenses

These lenses help make strong language safer before it becomes external output.

Possible lenses:

- claim-safety-level1
- dancing-trust-level1
- balance-direction-level1
- email-intent-checker-level1

Core principle:

a hard claim can often become an invitation to verify together.

### C — AI-building and prompt lenses

These lenses help people work with AI without confusing suggestions, actions and confirmed results.

Possible lenses:

- ai-output-status-level1
- cursor-prompt-safety-level1
- prompt-router-level1

Core principle:

proposal shared is not the same as executed, and executed is not the same as verified.

### D — Outreach and document lenses

These lenses help structure communication without turning it into pressure or sales language.

Possible lenses:

- outreach-log-level1
- email-sent-anchor-level1
- document-builder-level1
- powerpoint-outline-level1

Core principle:

communication should remain traceable, claim-safe and open to response.

### E — Living field lenses

These lenses help observe living systems without pretending to prove ecological or human intent.

Possible lenses:

- bird-observation-level1
- garden-field-level1
- pond-swimmable-level1
- living-field-observer-level1

Core principle:

observe visible movement, do not overclaim certainty.

### F — Local business and operational friction lenses

These lenses help small businesses and teams see repeated friction moments without heavy systems.

Possible lenses:

- bakery-demand-level1
- local-business-friction-level1
- csv-cluster-level1

Core principle:

make missed demand, repeated deviations or friction moments visible from simple local input.

### G — Family, school and personal interface lenses

These lenses must remain generic, private and local-only.

Possible lenses:

- family-interface-level1
- kids-symbol-board-level1
- school-choice-level1
- weekly-agenda-level1

Core principle:

support human clarity without extracting private life into a platform.

## Why this matters

Many systems try to solve friction by becoming heavier:

- more control
- more dashboards
- more automation
- more data extraction
- harder claims

AsymX Level 1 moves in the opposite direction.

It starts with a small local lens.

It does not take the data.

It does not replace the system.

It does not claim certainty too early.

It simply makes one hidden pattern visible.

## Shared rule

Do not fight the source.

Do not inflate the claim.

Do not upload private data.

Make the pattern visible, locally and safely.

## Invitation

If a lens is useful, inspect it.

If it can be improved, fork it.

If it reveals a real pattern, test it with mock or anonymized data first.

If internal data is involved, keep it local.

The field is open.

The source stays clean.
