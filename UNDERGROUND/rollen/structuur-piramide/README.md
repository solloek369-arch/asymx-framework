# Structuurpiramide

Doel: rollen ordenen van grond naar zichtbaarheid.

Niet als hiërarchie vanarde.
Wel als kijkhoogte / druklaag / verantwoordelijkheid.

## Laag 1 — Vloer / Directe werkelijkheid

Waar het fysieke veld spreekt.

Voorbeelden:
- picker
- chauffeur
- magazijnmedewerker
- vulploeg
- ontvangst
- retour
- balie
- klantcontact

Vraag:
Wat gebeurt er echt op de vloer?

## Laag 2 — Coördinatie / Dagsturing

Waar planning en uitvoering elkaar raken.

Voorbeelden:
- teamleider
- voorman
- planner
- shiftlead
- operationeel coördinator

Vraag:
Waar schuurt planning met werkelijkheid?

## Laag 3 — Systeem / Registratie

Waar software, voorraad, data en procesregels bepalen wat “waar” lijkt.

Voorbeelden:
- voorraadbeheer
- WMS / ERP beheer
- SAP / AS400 / systeembeheer
- data-analyse
- procesbeheer

Vraag:
Wat zegt het systeem, en klopt dat met de vloer?

## Laag 4 — Besluit / Management

Waar keuzes worden gemaakt over capaciteit, kosten, prioriteit en beleid.

Voorbeelden:
- operations manager
- logistiek manager
- category manager
- supply chain manager
- finance / control

Vraracte keuze veroorzaakt concrete druk?

## Laag 5 — Externe laag / Markt / Klant / Leverancier

Waar belofte, vraag, levering en verwachting binnenkomen.

Voorbeelden:
- klant
- leverancier
- transportpartner
- platform
- sales
- inkoop

Vraag:
Welke buitenbelofte wordt binnen als druk gevoeld?

## Gebruik

Elke case begint bij een situatie.

Niet:
“De picker doet X.”

Wel:
“Situatie: systeem zegt voorraad, vloer vindt leegte. Welke rollen zien welke asymmetrie?”
