# Sligro Rolpiramide — Intern

Status: UNDERGROUND  
Public status: NO by default  
Doel: rollen als lenzen op logistieke / operationele situaties.

Dit is geen officieel Sligro-document.
Dit is een interne AsymX-kijkstructuur op basis van publieke/algemene roltypen en operationele logica.

## Laag 1 — Vloer / Fysiek veld

Rollen als lens:

- orderpicker
- magazijnmedewerker
- reachtruckchauffeur
- ontvangstmedewerker
- expeditie
- chauffeur
- winkelvloer / zelfbedie/ emballage

Ziet vooral:

- klopt de voorraad fysiek?
- ligt het product waar het systeem zegt?
- is de pick uitvoerbaar?
- waar ontstaat wachttijd?
- waar ontstaat fysieke druk?

Typische asymmetrie:

Systeem zegt JA.  
Vloer zegt NEE.

## Laag 2 — Dagsturing / Teamlaag

Rollen als lens:

- teamleider
- shiftlead
- voorman
- dagcoördinator
- planner operatie

Ziet vooral:

- capaciteit vandaag
- mensen versus volume
- spoed versus rust
- planning versus verstoring
- wie moet waarheen bewegen?

Typische asymmetrie:

Planning lijkt rond.  
Dagwerk breekt open.

## Laag 3 — Voorraad / Systeem / Proces

Rollen als lens:

- voorraadbeheer
- procesbeheer
- WMS / ERP gebruiker
- data / rapportage
- replenishment
- inkoopondersteuning
- artikelbeheer

Ziet vooral:

- systeemvoorraad
- locatievoorraad
- mutaties
- tellingen
- correcties
- foutsporen

Typische asymmetrie:

Data is schoon op scherm.  
Werkelijkheid is rommelig op de vloer.

## Laag 4 — Management / Besluit

Rollen als lens:

- logistiek managerations manager
- vestigingsmanager
- supply chain manager
- finance / control
- HR / capaciteit

Ziet vooral:

- kosten
- output
- servicegraad
- personeelsdruk
- KPI’s
- structurele frictie

Typische asymmetrie:

KPI lijkt neutraal.  
Lichaam/vloer draagt de druk.

## Laag 5 — Klant / Leverancier / Markt

Rollen als lens:

- klant
- leverancier
- transportpartner
- sales
- accountmanager
- category / assortiment

Ziet vooral:

- beschikbaarheid
- leverbelofte
- assortiment
- prijs
- timing
- verwachting

Typische asymmetrie:

Buiten wordt iets beloofd.  
Binnen moet het lichaam het waarmaken.

## Sligro case-vraag

Gebruik deze piramide voor situaties zoals:

- ghost pick
- lege locatie, systeemvoorraad aanwezig
- telling klopt niet met vloer
- druk bij piekmoment
- orderbelofte versus fysieke capaciteit
- planning versus werkelijke route
- klantvraag versus interne voorraad

Altijd eerst:

Situatie → rollen als lenzen → asymmetrie → veilige zichtbaarheid.
