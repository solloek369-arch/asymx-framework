# GitHub as Public Connection Layer

## Purpose

GitHub functions as a **public routing and correction field** for AsymX.

It is not only code storage.

It is a **visible layer** where source, discussion, proposal, automation, review, and publication can remain traceable—without treating GitHub visibility alone as a correctness verdict.

---

## GitHub Surfaces

- **Repository:** public source tree and garden entrance  
- **Issues:** incoming signals, questions, friction, tasks  
- **Discussions:** open conversation and correction space  
- **Pull Requests:** visible proposed movement  
- **Actions:** automated checks and safety gates  
- **Projects:** visible routing board  
- **Codespaces:** repeatable shared build environment  
- **Pages:** optional public navigation interface  
- **Copilot / agents:** proposal layer only, never decision layer  

---

## AsymX Mapping

- **Issue** → signal / intake  
- **Label** → routing marker  
- **Branch** → safe movement space  
- **Pull Request** → visible proposal with diff  
- **Action / check** → guard  
- **Review / look** → human gate  
- **Merge** → confirmed public movement  

---

## Safety Boundary

- GitHub is **public by default** for public repositories.  
- Private source material does **not** move here directly.  
- Unpublished private-source material (including chat-style transcripts), local vault material, and identifiable people are **not** used as public proof.  
- AI agents may **propose** changes; they do **not** decide or merge.  
- Automated checks help, but do **not** replace human review.  
- **Diff** is stronger than statement.  
- No hard claims, no safety certainty claim, no outside-validation claim.

## First Practical Setup

1. Link the public mechanics index.  
2. Add this note.  
3. Enable Discussions manually in repository settings.  
4. Create issue templates for signal, correction, and proposal.  
5. Add routing labels such as `garden-ready`, `park`, `vault-boundary`, `claim-safety`, `mechanism`, `prompt`, `afterword`, `x-dance`, `needs-look`.  
6. Add a public safety scan with GitHub Actions.  
7. Add a devcontainer for Codespaces.  
8. Add GitHub Pages navigation later if useful.
