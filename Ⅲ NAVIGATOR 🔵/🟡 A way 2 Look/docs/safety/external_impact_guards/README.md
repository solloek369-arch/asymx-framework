# External-Impact Guards

Status: PUBLIC STRUCTURE

External-Impact Guards are AsymX safety mechanisms for situations where a temporary state, impulse, system error, or unclear context could create harm outside the user.

These guards do not label a person as bad.

They identify a high-risk field condition and activate a pre-committed protection rule.

## Examples of External Impact

- physical safety
- driving or vehicle access
- public communication
- financial decisions
- conflict escalation
- system commits or irreversible technical actions

## Core Pattern

High-risk context appears.

A pre-committed guard activates.

The risky action is removed, delayed, or routed through a safer gate.

## Privacy Rule

The public mechanism can be documented.

The private trigger remains private unless explicitly approved.
