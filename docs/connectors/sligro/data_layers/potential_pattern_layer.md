# Sligro — Potential Pattern Data Layer

Status: RESEARCH / MOVING TRACE  
Claim level: POTENTIAL / TO BE CONFIRMED  
Purpose: hold comparable logistics and sector patterns separately from Sligro facts

## Core principle

Not all useful data is Sligro data.

Comparable logistics data can still reveal patterns that may be relevant.

This layer exists for pattern-related data from:

- comparable logistics companies
- foodservice distribution
- warehouse operations
- ERP transitions
- floor-to-system mismatch cases
- public sector reports
- customer or supplier friction patterns
- general operational research

This data must never be presented as Sligro fact.

## Different color

This layer needs a different color or label from public Sligro data.

Suggested label:

POTENTIAL PATTERN DATA

Meaning:

- possibly relevant
- not confirmed inside Sligro
- useful for comparison
- waiting for verification
- not a claim

## Why this layer matters

Patterns often become visible outside one organization first.

A repeated logistics issue in a comparable environment may help ask better questions inside Sligro later.

The value is not proof.

The value is direction.

## Confirmation needed

Every potential pattern should be treated as a question:

- Could this happen here?
- Does this match floor reality?
- Is this already solved?
- Is this irrelevant?
- Does this need internal confirmation?
- Does this need to be parked?

## Claim rule

Potential data cannot become Sligro data until Sligro or an authorized internal signal confirms it.

Core sentence:

Pattern is not proof.  
Comparison is not confirmation.  
Potential data asks better questions.  
Sligro confirms what is Sligro.
