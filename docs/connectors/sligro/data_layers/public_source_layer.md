# Sligro — Public Source Data Layer

Status: RESEARCH / MOVING TRACE  
Claim level: PUBLIC SOURCES ONLY  
Purpose: collect and structure public Sligro information as the safe source layer

## Core principle

The first Sligro data layer must be built only from information that is already public.

Public means:

- published by Sligro
- published in annual reports
- published in official press releases
- published on public company pages
- publicly verifiable through reliable sources

This layer does not claim internal access.

This layer does not claim hidden problems.

This layer does not claim agreement from Sligro.

It only structures what is already visible.

## Why this layer comes first

Public Sligro data is the safest starting point because it is already visible from Sligro itself or from public reporting.

This allows AsymX to build a visible source base without inventing claims.

The public source layer protects the work from speculation.

## Possible public source categories

The public source layer may include:

- annual reports
- half-year reports
- official press releases
- public strategy statements
- public financial figures
- public logistics and operational context
- public IT transition context
- public sustainability and people statements
- public company structure
- public location and service information

Every item should keep its source reference.

## Public transition context

Public sources may describe Sligro's movement from legacy ERP context toward SAP-based structures.

This can be used only as public context.

It must not be used as proof of internal problems, failure, blame, or hidden loss.

## Claim rule

A public source can support only what it actually says.

No public source may be stretched into an internal claim.

Core sentence:

Public first.  
Source visible.  
No hidden claim.  
No internal conclusion without internal confirmation.
