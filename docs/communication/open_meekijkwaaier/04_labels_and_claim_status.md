# Labels And Claim Status

## Purpose

This document defines the Open Meekijkwaaier labels.

Labels keep visibility separate from claim.

## Label Set

### OBSERVATION

Something visible enough to name.

Safe phrasing:

- We observe this signal.
- This appears in the visible material.

Unsafe phrasing:

- This proves what happened.
- This is the internal truth.

### EXAMPLE

A concrete case used to make a pattern easier to inspect.

Safe phrasing:

- This example can help us look together.
- This example may make the question clearer.

Unsafe phrasing:

- This example proves the whole pattern.
- This example is the final answer.

### HYPOTHESIS

An AsymX interpretation that remains open for correction.

Safe phrasing:

- One possible reading is...
- This may indicate a reviewable direction.

Unsafe phrasing:

- This is the cause.
- This is what is happening inside.

### QUESTION

An open route for shared review.

Safe phrasing:

- What should be checked here?
- What is missing from this view?

Unsafe phrasing:

- This question already proves the problem.
- The answer is already known.

### PATTERN

A repeated or connected signal that may be useful to review.

Safe phrasing:

- This may be a pattern.
- This can be reviewed as a repeated signal.

Unsafe phrasing:

- This pattern guarantees the conclusion.
- This pattern proves failure.

### ROUTE

The next safe movement for the visible material.

Safe phrasing:

- Route to correction.
- Park for later.
- Review with the relevant role.

Unsafe phrasing:

- Move directly to solution.
- Decide without review.

### DO NOT CLAIM

A boundary marker for what should not be said.

Safe phrasing:

- Do not claim internal cause.
- Do not claim proof.
- Do not claim final conclusion.

Unsafe phrasing:

- This is confirmed.
- This is the root cause.
- This proves failure.

## Rule

More visibility is allowed when claim status is clear.

If claim status is unclear, keep the material open, label it or park it.
