# Prompt Opening Translation Protocol

Status: PUBLIC CANDIDATE  
Layer: Prompt Architecture / Source Translation / Afterword Trace

## Core Insight

In AsymX, a prompt is not only an instruction.

A prompt is also an opening signal.

Different humans and AI systems may read the same prompt through different assumptions, training, language habits, cultural frames, or technical defaults.

Therefore, important prompts should not be stored only as raw commands.

They should be translated into layered traces.

## Why This Exists

Raw language can carry strong source movement while still being imprecise.

If an AI answers only the surface wording, it may speculate, overfit, moralize, simplify, or respond from the wrong frame.

The goal of this protocol is to preserve the source signal while also creating a clearer, safer, more transferable mechanism.

## Prompt Layers

Each important prompt may contain:

1. Raw Prompt
The original words as intact as possible.

2. Loek Translation
A direct translation of what the prompt is trying to open, in plain project language.

3. Source Translation
A claim-safe mechanism translation that can be understood outside the original emotional, personal, or situational context.

4. Non-Speculation Boundary
What the response should not assume.

5. Desired Response Direction
The kind of answer or action the prompt is trying to invite.

6. Routing
Whether the mechanism belongs in public GitHub, local vault, or private/personal storage.

## Raw Prompt

The raw prompt is a trace.

It should not be overcorrected too early.

The raw version preserves movement, urgency, tone, and emergence.

## Loek Translation

The Loek translation answers:

- What is being opened?
- What is Loek trying to make visible?
- What does the prompt point toward?
- What should not be flattened into generic AI language?

## Source Translation

The source translation answers:

- What is the reusable mechanism?
- What is the claim-safe version?
- What can another AI or human understand without knowing the full background?
- What can be public without exposing private material?

## Afterword Layer

An afterword records what became visible after the prompt was used.

It may include:

- what the prompt opened
- what changed in the system
- what became clearer
- what should be corrected
- what belongs in public
- what must remain private
- what new rule, path, or wording appeared

## Public / Private Rule

Public GitHub may carry the mechanism.

Local vault carries private source, raw personal context, names, sensitive traces, and unfinished emotional material.

## Response Rule for AI

Do not answer only the surface words.

First translate the opening.

Then answer the mechanism.

If the prompt contains private or sensitive material, separate the reusable public pattern from the private trace.

## Closing Line

A prompt opens a field.
A translation makes the opening readable.
An afterword records what the field returned.
