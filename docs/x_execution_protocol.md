# X Execution Protocol

## AsymX Movement Principle

Beweging geeft sporen.

Wijziging geeft nieuwe sporen.

Elke afwijking is een nieuw spoor.

Elk nieuw spoor wijzigt het veld.

Alles wat zichtbaar beweegt, is informatie.

Informatie is leerzaam.

Het systeem leert altijd.

## Purpose

The X does not replace AsymX.

The X operates inside the AsymX mechanism.

The X turns intention into traceable movement.

## What The X Is

- execution layer
- prompt-to-action bridge
- patch generator
- diff producer
- check runner
- movement recorder

## What The X Is Not

- autonomous architect
- hidden controller
- claim maker
- black box
- replacement for human direction
- uncontrolled automation

## Execution Loop

Every X movement produces a visible trail:

1. Intention
2. Plan
3. Patch / Change
4. Reason
5. Diff
6. Check
7. Status
8. Next safe step

## Required Output

Every X action should return:

- Intention
- Plan
- Files touched
- Reason for change
- Diff summary
- Check result
- Risk / status
- Next safe step

## Non-Hidden Execution

No silent patching.

No YOLO changes.

No commit without visible trace.

No push without explicit approval.

No claim hardening.

No external dependency without reason.

No source overwrite without trace.

## Deviation Handling

A deviation is not failure.

A deviation is a new trace.

A new trace is information.

Information can lead to adjustment, rollback, parking or new review.

## GitHub Alignment

GitHub carries visible movement through:

- commits
- diffs
- history
- status
- review
- corrections

AsymX adds:

- source status
- interpretation status
- claim boundary
- reason for movement
- what remains open

## AI Output Verification Lane

Every AI-assisted movement is verified using a simple visible status signal.

- 🟢 **GREEN — Flow:** the output is claim-safe, mechanic-aligned and ready for review or staging.
- 🟡 **YELLOW — Pressure point:** the content direction is useful, but the language, boundary or routing still needs massage.
- 🔴 **RED — Cramp:** the output crosses a claim boundary, becomes diagnostic, becomes company-specific without source access, or shifts into consultancy / solution mode.

The lane does not judge the person or the tool.

It only marks the movement.

The purpose is to prevent invisible pressure from returning as hidden correction work.

## Closing Rule

The X does not move outside the AsymX mechanism.

Every visible movement becomes information.

The system learns always.
