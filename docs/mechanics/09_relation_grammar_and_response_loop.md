# Relation Grammar and Response Loop

Mechanic 07 defines the source-jump:

> public trace → question-first lens → source-jump → open invitation

Mechanic 09 defines the operational grammar after that jump.

This file does not create outreach.

This file does not authorize action.

This file does not turn questions into claims.

It defines how AsymX keeps source-jumps consistent, traceable, and claim-light.

## 1. Core Units

### Node / Source

A node is a visible trace.

It may be a public link, document, statement, dataset, page, policy, pattern, or recorded source pointer.

A node does not become AsymX property.

### Lens

A lens is the question placed between sources.

The lens does not declare meaning.

The lens asks what possible relation appears.

Allowed form:

> What possible tension, mismatch, transition, pressure, direction, or recovery path appears when Source A and Source B are placed in relation?

### Jump

A jump is the routed relation between two or more nodes.

A jump is not a conclusion.

A jump is a structured question.

### Invitation

An invitation is the open ending of a jump.

It asks whether the relation is worth exploring.

It does not require agreement.

### Tentacle

A tentacle is a temporary landing field.

A tentacle is not the core.

A tentacle is not proof.

A tentacle is not an owned domain.

## 2. Allowed Tension Types

A source-jump may label the possible tension type, but only as a question or candidate.

Allowed candidate types:

- friction
- mismatch
- delay
- overload
- underload
- transition
- reversal
- blind spot
- recovery path
- routing gap
- ownership gap
- attention gap
- trust gap
- pressure build-up
- pressure release
- unresolved relation

Rule:

Do not write:

> This is friction.

Write:

> What possible friction or mismatch appears here?

Or:

> Candidate tension type: mismatch / transition / recovery path.

## 3. Allowed Output Forms

A source-jump may produce only these output forms:

1. **Question**  
   Opens observation.

2. **Invitation**  
   Asks whether exploration is wanted.

3. **Route**  
   Points to another source, file, or next safe step.

4. **Candidate label**  
   Names a possible tension type without turning it into fact.

5. **Micro-validation proposal**  
   Suggests the smallest contained way to test a relation, only after a human gate.

6. **Park marker**  
   Marks the relation as not-for-now.

7. **Vault marker**  
   Marks the relation as private or not public.

## 4. Pressure State

Every active source-jump may carry a pressure state.

### HOT

Possible immediate cost, time loss, blockage, overload, or active pressure.

Output style:

small, contained, pressure-reducing question.

### WARM

Possible recurring pattern or operational drag.

Output style:

lightweight observation and next source pointer.

### COLD

Long-term structural mismatch, lost potential, or slow degradation.

Output style:

parkable question, research pointer, or future trail.

### BLACK

Private, unclear, too personal, unsafe, or not ready.

Output style:

vault / stop / human review.

Pressure state is not proof.

It is a routing signal.

## 5. Response Loop

After an invitation, the field may respond in different ways.

### Acceptance

Meaning:

The field is willing to explore.

AsymX move:

Do not expand into a large project.

Allowed next step:

Micro-validation only.

Micro-validation means:

the smallest contained no-harm test that can observe whether the relation is useful.

It may be:

- one manual check
- one short observation window
- one small comparison
- one contained walkthrough
- one reversible test

It is not:

- a full implementation
- a system integration
- a budget request
- a claim of proof
- permission to bypass the human gate

### Rejection

Meaning:

The field does not accept the invitation or does not recognize the relation.

AsymX move:

Do not argue.

Do not escalate.

Do not turn rejection into proof.

Allowed next step:

Label as response trace.

Possible label:

> Response trace: relation not accepted at this time.

The conclusion remains with the observer.

### Silence

Meaning:

No response is available.

AsymX move:

Do not chase by default.

Do not intensify automatically.

Do not treat silence as proof.

Allowed next step:

Park the invitation.

Possible label:

> No-response trace: relation parked until new source, signal, or human gate appears.

## 6. Forbidden Moves

Do not write:

- This proves the system is wrong.
- This reveals the real problem.
- The field is blind.
- The organization denies friction.
- The public sources clash.
- AsymX cannot fail.
- This must be implemented.
- Silence proves pressure.
- Rejection proves the lens.
- This tentacle is the system.

## 7. Minimal Source-Jump Format

Use this format for repeatability:

```text
Source A:
Source B:
Relation question:
Candidate tension type:
Pressure state:
Tentacle invited:
Output form:
Visibility:
Next safe step:
Human gate required:
```

Human gate remains required before outreach, test, commit, or public expansion.
