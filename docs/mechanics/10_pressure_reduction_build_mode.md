# Pressure Reduction Build Mode

## Simple meaning

Not every signal must become a task, file, commit or conversation.

Pressure is lowered by letting signals exist first, before deciding what they become.

## Core separation

1. Capture is not execution.
2. Phone mode is not laptop mode.
3. Private source is not public mechanism.
4. Recognition is not publication.
5. Idea signal is not deep conversation.

## Three front doors

### Idea Box

Fast signal capture.

No analysis required.

No conversation required.

Used when presence matters more than depth.

### Discussion

Short reflection or clarification.

Small answer only.

No deep build loop unless explicitly opened.

### Extended

Full analysis, system design, file creation, GitHub preparation or deeper review.

## Public mechanism rule

Only the mechanism may go public by default.

Source material stays private unless separately reviewed.

## X behavior

X may recognize:

- this is a public-safe mechanism candidate
- this belongs in private source
- this needs 2 Look
- this is too hot for publication
- this is ready for later build mode

X must not automatically:

- widen scope
- publish raw context
- commit or push
- convert every idea into action
- treat urgency as permission

## Pressure reduction formula

Signal first.

Weight later.

Build only after gate.

## Safe next step

Keep the first version small.

Do not solve the whole system here.

Create the pressure-reduction mechanism note only.
