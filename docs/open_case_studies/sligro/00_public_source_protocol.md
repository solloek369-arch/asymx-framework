# Public Source Protocol — Sligro Open Source Reviews

## Status

PUBLIC-SOURCE REVIEW / OPEN-SOURCE VISIBILITY LENS

These documents use only publicly visible information.

They do not use:
- internal Sligro data;
- private communication;
- employee reports;
- confidential documents;
- internal measurements;
- operational claims from inside the company.

The purpose is not to diagnose Sligro.

The purpose is to show how public signals can be translated into claim-safe AsymX visibility questions and small validation patterns.

## Core Separation

Every statement in this folder must remain traceable to one of three layers:

1. Public source
2. General sector pattern
3. Explicitly marked AsymX hypothesis

These layers must not be mixed.

## Allowed Language

Use:

- Public sources describe...
- This publicly visible signal can be read as...
- In logistics operations, this pattern may become visible...
- This is researchable through existing exports...
- This may invite shared validation...
- This is visible enough to review together...
- Based on publicly available information only...

## Not Allowed Language

Do not use:

- Sligro has...
- Sligro proves...
- The system fails...
- The floor must repair system errors...
- This proves...
- This guarantees...
- Internal problems exist...
- Root cause is...
- The operation suffers from...

## Case Framing

A Sligro document in this repository is not an internal case study.

It is a public source review.

Better framing:

- Public Signal Review
- Public Source Review
- Open Visibility Lens
- Claim-Safe Review

Avoid framing that suggests internal access, diagnosis, audit, consultancy or implementation.

## Source Rule

If a company name is used, the public-source status must be visible nearby.

If an interpretation is added, it must be labeled as AsymX interpretation or hypothesis.

If a claim cannot be traced to a public source, it must be rewritten, parked or removed.

## Movement Rule

When the repository changes direction, the movement should be visible.

A correction is not damage control.

A correction is a visible calibration step.

AsymX does not hide movement.

AsymX shows why the movement happened.

## Closing Rule

If it is publicly visible, it may be used.

But only when the public source, the AsymX interpretation and the claim boundary remain visibly separated.
