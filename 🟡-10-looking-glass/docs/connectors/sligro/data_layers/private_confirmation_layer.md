# Sligro — Private Confirmation Layer

Status: RESEARCH / MOVING TRACE  
Claim level: PRIVATE / ROLE-BASED / CONSENT-BASED  
Purpose: define how internal confirmation could work without hidden monitoring or personal surveillance

## Core principle

Inside Sligro, the richest data can only open safely through permission, trust, role access, and purpose limitation.

This layer is not public.

This layer is not employee surveillance.

This layer is not hidden monitoring.

It is a possible private confirmation space where authorized people can confirm, correct, enrich, or reject visible patterns.

## Click as responsibility signal

A click can be a small act of responsibility.

Examples:

- this happens
- this does not happen
- this happens often
- this location is wrong
- the system says A, the floor sees B
- this causes delay
- this prevents a mistake
- this needs attention
- this is already solved
- this is not relevant

The click is not an accusation.

The click is a trace.

## Low-pressure input

The confirmation layer should make it easy for people to contribute without social pressure.

Useful design principles:

- small buttons
- no blame language
- no forced explanation
- optional context
- clear purpose
- visible feedback
- role-based access
- aggregated pattern view
- correction possible
- stop and park options

## Privacy and safety

This layer requires:

- consent
- role-based access
- minimal data
- purpose limitation
- no hidden tracking
- no unnecessary personal identification
- safe aggregation
- clear visibility rules
- correction rights
- human governance

A person should not become the product.

The pattern should become visible.

## Public output boundary

Private confirmation does not automatically become public.

Only aggregated, claim-safe, approved patterns may move outward.

Raw internal signals stay protected.

## Claim rule

Internal confirmation is the only layer that can turn potential pattern data into Sligro-specific data.

Core sentence:

A click is a trace, not blame.  
A pattern opens only with permission.  
Private confirmation protects people.  
Only approved patterns move outward.
