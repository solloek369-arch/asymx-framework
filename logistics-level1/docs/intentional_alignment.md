# Intentional Alignment — AsymX Logistics Level 1

This project is not marketing.

It is designed to be found by people who are already searching for this type of operational friction.

The goal is not to persuade.
The goal is to reduce findability friction.

## Core principle

Help the seeker, not the buyer.

A warehouse manager, WMS analyst, supply chain lead or IT reviewer should be able to find this project through the words they already use when describing the problem.

## The three-layer intent model

### 1. Head terms open recognition

Examples:

- ghost pick
- manco
- short pick
- warehouse deviation
- inventory mismatch
- WMS mismatch
- voorraadverschil
- herhaalde manco

These terms help people recognize the broad problem.

### 2. Depth terms prove practical usefulness

Examples:

- CSV schema
- WMS export
- pick-log example
- location_id
- item_id
- event_type
- timestamp
- repeated manco per location
- short-pick cluster
- shadow run template

These terms help serious searchers recognize that the project is practical and inspectable.

### 3. Trust terms reduce adoption friction

Examples:

- offline analyzer
- browser-only
- no data upload
- no backend
- no dependencies
- mock data
- security policy
- checksums
- no real company data

These terms help IT, security and operations teams verify that Level 1 can be reviewed safely.

## Key sentence

Head terms attract attention.
Depth terms prove usefulness.
Data structure creates trust.

## AsymX interpretation

The repository is not only a demo.
It is a visible technical source layer.

People who search broadly can understand the tool.
People who search deeply can inspect the mechanism.
People who need safety can verify the no-data model.

## Do not use

- best solution
- revolutionary platform
- guaranteed savings
- proven company loss
- sales demo
- request access to see value

## Use

- static local analyzer
- repeated deviation cluster
- candidate for shadow-run review
- WMS export CSV schema
- no upload
- browser-only
- mock data
- claim-safe output
