# AGENTS.md

## Cursor Cloud specific instructions

### Overview

This is the **AsymX Framework** — a collection of static, self-contained browser-based tools ("Level 1 lenses") plus one Python CLI utility. There is **no build step, no package manager, no external dependencies**.

### Components

| Component | Path | How to run |
|---|---|---|
| Logistics Level 1 | `logistics-level1/index.html` | Open in browser (via file:// or HTTP server) |
| Chat Intake Level 1 | `chat-intake-level1/index.html` | Open in browser |
| Story Filter Level 1 | `story-filter-level1/index.html` | Open in browser |
| X-Loop Engine | `tools/x_loop/x_loop_engine.py` | `echo "input text\nEND" \| python3 tools/x_loop/x_loop_engine.py` |

### Running the HTML lenses

Start a local HTTP server from the repo root:

```bash
python3 -m http.server 8080 --directory /workspace
```

Then access lenses at:
- http://localhost:8080/logistics-level1/index.html
- http://localhost:8080/chat-intake-level1/index.html
- http://localhost:8080/story-filter-level1/index.html

### Running the X-Loop Engine

The Python script uses only stdlib (no pip install needed). Pipe input ending with `END`:

```bash
printf "some input text\nEND\n" | python3 tools/x_loop/x_loop_engine.py
```

It writes output to `tools/x_loop/NEXT_X_PROMPT.md` and logs to `tools/x_loop/logs/x_movement_log.jsonl`.

### Linting

No linting tools are configured. For Python, use `python3 -m py_compile` for syntax validation. HTML files are self-contained with inline CSS/JS and no build tooling.

### Key caveats

- The HTML lenses have strict Content-Security-Policy headers blocking all network requests (`connect-src 'none'`). They are designed to work fully offline.
- There is no `requirements.txt` or `package.json` — this is intentional by design.
- The `.gitignore` is a standard Python template auto-generated by GitHub; it does not reflect actual Python project complexity.
- Generated files from `x_loop_engine.py` (`NEXT_X_PROMPT.md`, `logs/`) should not be committed.
